@nex-era ให้คำปรึกษาด้านความเป็นผู้นำ ด้านธุรกิจและองค์กร นอกจากนี้ยังทำงานด้านเทคโนโลยี โดยเฉพาะการออกแบบซอฟท์แวร์ การพัฒนา รวมถึงบริการอื่นๆอีกหลายอย่าง

#UI
User Interface is the heart of every software

ทุกอย่างที่ผู้ใช้งานต้องอยู่กับมันตลอดเวลาก็คือ หน้าจอ (หรือภาษาอังกฤษว่า User Interface) นี่แหละ ในเรื่องของการออกแบบและพัฒนาซอฟท์แวร์นั้น มีหลายอย่างที่ต้องคำนึงถึง แสดงคร่าวๆดังนี้
  • หน้าจอ หรือ UI (User Interface)
  • การจัดการข้อมูล หรือ Data Modeling
  • ขั้นตอนการทำงาน (Process)
  • การจัดการ user
  • ฯลฯ

#Easy
Easy-to-use is key success factor

ซอฟท์แวร์ที่ดีต้องใช้งานง่าย คือใช้ได้ทันที ไม่จำเป็นต้องมาฝึกอบรมอะไรให้วุ่นวาย เหมือนเราเข้าไปในร้านอาหาร แล้วเปิดเมนู แล้วก็สั่งได้เลย ความง่ายของซอฟท์แวร์ก็ต้องประมาณนั้น

ภาพนี้เป็นตัวอย่าง sketch board เพื่อร่างหน้าตาของซอฟท์แวร์ที่เราต้องการ อาจจะเรียกว่าเป็นขั้น idea ก็ได้

#Data
การออกแบบโครงสร้างข้อมูล (Data Model) และการจัดการ database เพื่อให้เราได้ข้อมูลที่ครอบคลุม เรื่องนี้เป็นเรื่องที่ละเอียดอีกเรื่องหนึ่ง เวลาเราจะออกแบบหรือพัฒนาซอฟท์แวร์

#Process
Process is like backbone

ขั้นตอนการใช้งานก็ดี การทำงานก็ดี จะต้องศึกษาอย่างละเอียด แล้วออกแบบให้เหมาะสม พยายามหลีกเลี่ยงอย่าให้คน คิด คำนวน หรือกด อะไรมากเกินไป เอาเฉพาะที่จำเป็น ทำโปรแกรมให้อำนวยความสะดวกให้มากที่สุด เพื่อลดความผิดพลาด และเพื่อความปลอดภัยของข้อมูลไปในตัว

#User
User handling is very detail task

ความสัมพันธ์ของ user ไม่ว่าจะใช้งานคนเดียว เป็นทีม หรือเป็นองค์กรใหญ่ๆ ยิ่งคนใช้เยอะ ก็ยิ่งต้องละเอียดมากเท่านั้น ใครมีบทบาทอะไร ทำอะไรได้แค่ไหน เป็นต้น

#Tech
Dev language, database, cloud, ...

ทางด้านตัวเทคโนโลยี หรือเครื่องมือที่ต้องใช้ ก็มีเรื่องต้องคิดหลายเรื่อง เช่นเราจะใช้แบบ web หรือแบบ mobile app จะใช้ภาษาอะไรมาโปรแกรม ใช้ database อะไร รวมไปถึงเครื่อง hardware อะไรต่างๆ ก็เป็นเรื่องละเอียดอ่อนเช่นกัน ตัวอย่าง technology ที่ทางเราใช้ หรือเป็นที่นิยมกัน เช่น java script, node.js, java, c#, python, mongodb, mysql, postgresql, linux, windows, cloud และอะไรอีกเยอะแยะ
